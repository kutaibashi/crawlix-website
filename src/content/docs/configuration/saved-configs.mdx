## What Are Saved Configurations?

A saved configuration is a named snapshot of your crawl settings: max pages, depth, concurrent workers, URL filters, JS rendering settings, and more. You can save a configuration and reload it later when crawling the same (or a different) site.

## Saving a Configuration

1. Configure your crawl settings as desired.
2. Click **Save Configuration** (in the settings panel or via the menu).
3. Give the configuration a descriptive name (e.g., "Full crawl — production", "Blog only — weekly", "Mobile JS rendering").

Configurations are associated with the domain of the start URL by default, but you can apply any saved configuration to any domain.

## Loading a Saved Configuration

1. Enter the site URL you want to crawl.
2. Click **Load Configuration** or select from the configuration dropdown.
3. All settings are restored from the saved configuration.
4. Start the crawl.

## Managing Configurations

Access your saved configurations at **Settings > Saved Configurations**. From there you can:

- View all saved configurations
- Edit any configuration
- Duplicate a configuration as a starting point for a new one
- Delete configurations you no longer need

## Use Cases

- **Consistent audits** — run the same settings monthly to ensure comparable results over time
- **Multiple audit types** — one configuration for a full crawl, another for blog-only, another for mobile simulation
- **Client sites** — agencies can save a standard configuration for each client type (ecommerce, blog, corporate, etc.)
