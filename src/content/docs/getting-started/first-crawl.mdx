## Starting a Crawl

1. **Enter the URL** — type or paste the site's homepage URL into the address bar at the top of the app (e.g., `https://example.com`).
2. **Review settings** — Crawlix uses sensible defaults, so you can click **Start Crawl** immediately. Or adjust settings first (see below).
3. **Click Start Crawl** — the crawl begins immediately. A progress bar and live counter show how many pages have been discovered and processed.

## Optional Settings Before Crawling

Before starting, you may want to configure:

- **Max pages** — Free tier: 100, Pro: 50,000, Agency: unlimited. Only HTML pages count toward this limit.
- **Max depth** — how many link levels to follow from the start URL. Default is 10.
- **JavaScript rendering** — enable this for React, Vue, Angular, or other JavaScript-heavy sites.

## During the Crawl

Crawlix shows real-time progress including:

- Pages discovered and pages crawled
- Current page being processed
- Issues found so far (by severity)
- Estimated time remaining

You can pause and resume a crawl at any time using the **Pause** button.

## When the Crawl Completes

Once finished, a summary screen appears showing:

- Total pages crawled
- Issue counts by severity: Errors, Warnings, and Notices
- Crawl duration

Click **View Issues** to dive into actionable findings, or navigate to any view from the left sidebar.

## Page Limit Behavior

When Crawlix reaches the page limit, the crawl stops automatically. The data collected up to that point is fully usable — you can review issues, export data, and compare results even from a partial crawl. Upgrade to Pro or Agency to crawl larger sites.
